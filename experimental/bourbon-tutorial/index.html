Getting started with Bourbon (without Rails)


Hello there! Have you been looking for a walktrhough style tutorial for Bourbon? This tutorial will walk you through the steps you need to have all the Bourbon tools with a little bit of boiler plate/starter code. Most of the tutorials I found for Bourbon were tied to Rails. This tutorial is for use of Bourbon wihtout Rails.
Bourbon is a front-end/css framework that will help you with vendor prefixes(write css without having to write -webkit... for crossbrowser compatability), make responsive grid layout for your components, give your elements a modern and cohesive apperance that can be easily changed, and common modern components and patterns (like accordian/tabs, cards, hero unit, dropdown menus and much more).
Bourbon is focused on being lightweight and is thus broken up into 5 components and requires a little configuration out of the box.
Prerequisites:
  • A fundamental understanding of html
  • A fundamental understanding of css
  • Basic ability to use terminal/command-line
  • Programming basics (know the following terms: function, variable, string), for refills/empties a basic knowledge of jQuery is also a prerequisite

Quick overview: Bourbon eponymous - Bourbon provides vendor prefixes where they are needed. Bourbon had more features, but they have been whittling them down and moving them to other parts. It is also required for bitters, refills and empties
Neat - doesn't require Bourbon. it is a semantic grid framework. The great thing about this grid framework is that it puts the grid into the css rules and not into classes(compare to bootstrap). This lets you keep your classes and ids relevant to your content, and lets the scss handle the appearnace. Does this make total sense? YES- To me it does. No cluttering up the html with class names that represent what the css looks like.
Bitters requires Bourbon . Bitters loads a number styles for html elements, immediately giving a modern look to your html. Because of its minimalist nature, it is designed for you to build up your custom styles on top of it.
Refills & Empties Requires Sass, Bourbon & Neat (and sometimes jQuery -depending on the component). Once you have installed those, you go to Refills or Empties and copy paste the code of the component you want into your own code. WAT. No, but really. And it's awesome. Take what you need, no less and no more philosophy. Lightweight, and totally customizable, you get awesome tools and total control. Back to top

Preparation

In the next parts I am going to walk you through the steps to set up and quickly confirm that you have set up your code correctly. You can either use it to start your own project, or I may develop a little project to continue to work through, if you are the kind of person that learns by doing projects

  1. Create a new directory
  2. In that dirtory make 2 more directories: css, js
  3. make file called index.html
  4. inside the css directory, create main.scss
  5. inside the js directory, create app.js
  6. open the project in your favorite editor (I use atom)
  7. Do it in code: 
    
                mkdir bourbon-project
                cd bourbon-project
                mkdir css
                mkdir js
                touch index.html
                touch css/main.scss
                touch js/app.js
                atom .
  8. if you are using atom, don't forget you can type html and then press tab and the boilerplate html will show up
  9. in your js file, write an alert 
    alert('js linked');
  10. right above your body tag, include a script tag linking to your js tag
  11. open index.html in the browser
  12. check the alert, if it works, go into app.js and delete the alert
  13. if it doesn't work, check your routes, get them working and then delete that alert! It'll get annoying if you don't

all of the above should have been familiar, if you had the suggested pre-reqs

Configure Sass

  1. Install Sass, if you have not already
  2. Not sure if you have Sass? Check by running
     sass -v
  3. Open a new tab in your terminal (with settings/current location- which should be the route of your bourbon-project)
  4.  sass --watch css/main.scss
  5. The above command will create the files css/main.css and css/main.css.map additionally, it will listen for changes to the main.scss file, so you should keep this tab open and have this script running when you are working on your main.scss file

      6. Write your first Sass

    1. in main.scss 
      
                  $favorite-color:#265b6a;

                  body{
                    background-color: $favorite-color;
                  }
    2. don't forget to save your file
    3. check that your Sass was compiled into your main.css file - it should look like 
      
      body {
        background-color: #265b6a; }

      /*# sourceMappingURL=main.css.map */
    4. link the main.css stylesheet in your index.html
    5. make sure all your files are saved
    6. check out your index.html in your browser. The background should now be your favorete color!
    7. This barely begins to demonstrate the amazingness of Sass! But we must press onwards because this tutorial is for Bourbon!

Grab Bourbon

    Install bourbon
  1. be sure you are in the css folder of your project
  2. gem install bourbon
  3. bourbon install
    in your css directory for the project
  4. you should now see a new directory in your css directory called 'bourbon'
  5. install bourbon into stylsheets by typing 
     @import "bourbon/bourbon";
    at the top of the stylesheet
  6. also 
     bourbon help
    and 
     bourbon update
  7. main.css should not have changed
  8. Check if bourbon is installed properly- update main.scss to match this 
    
            @import "bourbon/_bourbon.scss";

        $favorite-color:#265b6a;
        $text-color: black;

        body{
          background-color: $favorite-color;
          color: tint($text-color,75%);

        }
    to the body element in your main.scss
    now your main.css should be 
    
        body {
          background-color: #265b6a;
          color: #bfbfbf; }

Make it Neat

    Install Neat
  1. be sure you are in the css folder of your project
  2. gem install neat
  3. gem update sass
  4. neat install
    in your css directory for the project
  5. you should now see a new directory in your css directory called 'bourbon'
  6. install bourbon into stylsheets by typing 
     @import "neat/neat";
    at the top of the stylesheet
  7. if all went well, your main.css file should now look like this: 
    
  html {
    box-sizing: border-box; }

  *, *::after, *::before {
    box-sizing: inherit; }

  body {
    background-color: #265b6a;
    color: #bfbfbf; }

  /*# sourceMappingURL=main.css.map */

Add Bitters

    Install Bitters
  1. in your css directory
  2. Download and install normalize.css
  3. Add normalize.css to your index.html before your main.css
  4. gem install bitters
  5. bitters install
  6. add bitters to main.scss
  7.  @import "base/base";
    at the top of the stylesheet
  8. This should generate a lot of css in your main.css file and when you reload index.html you should see the updated styles of your html

Get Refills

    Refills are not installed

  3. click show code, then you copy and paste each component, (html, scss and/or javascript). If the code will go in the respective files, the javascript will go in the app.js file we set up at the beginning of this tutorial.
